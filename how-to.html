How to use this site:

This page explains how to navigate and use the features of the Exploring Opportunity site that visualized the Equality of Opportunity data from the Chetty et al. (2014) study identifing the causal effects of geography on social mobility.
This site was developed and tested using Chromium Browser (Google Chrome). If something looks or acts a bit off, it might be because your browser.

Navigation:

Each of the options menus can be closed and opened by clicking on the bars on the edges of the window like the one below (the colors are reversed for contrast).

Zoom:

Click to zoom and center on a geography. To get a closer look, click on a smaller geography; the map scales proportionally to the area of the geography. You can recenter the map by clicking the same geography twice or by clicking on the background of the map.

Using your own data:

This site allows you to open any additional data that you’d like to explore that isn’t already in the dataset. These files will not be saved or uploaded to a server anywhere. To prevent data loss due to differences in data types and handling across software and languages, I suggest saving all of you data (at the very least the geographic identifiers) that you intend to use as UTF-8 encoded character stings before attempting to load it.
Variables should not be stored as null unless you expect them to be treated as zeros or they are stored as a character sting and you expect them to treated as missing values. Variables in this form have not been rigorously tested; you may risk data loss and failed or incorrect merges.
If any of your variables have the same name as a variable in the EO dataset, the EO data will be overwritten.
  1. Click the top navigation bar at the top of the page, above the map—a dropdown box will open.
  2. Click
    Choose a .csv
    . Your machine will open a dialog box and allow you choose a file that will open in your machine’s memory.
  3. Once the file is successfully opened, the rest of the menu will be become active. Click on the selector at the top of the third column and choose the variable in your dataset that uniquely matches the geographic identifier in the Equality of Opportunity dataset. It must be a concatenation of the 2000 two-digit state FIPS and three-digit county FIPS codes with leading zeros.
    If you are loading data with missing geographic identifies, they should be stored as NA or NaN.
  4. Upon completion, you should see some information about the success rate of the merge.
    There are two ways for your merge to partially or completely fail: your data contains codes not found in Equality of Opportunity data or vice versa.
    If your merge partially (or completely) failed, you can download a .json file that the unmatched observations from your data set and the identifiers of the unmatched observations from the EO data. A generalized example of the file structure is below:
    {
    unmatched_cli:
     [{var1: observation1, var2: observation1, ... varK: observation1},
     {var1: observation2, var2: observation2, ... varK: observation2},
     ...,
     {var1: observationN, var2: observationN, ... varK: observationN}],
    unmatched_serv:
     [{GEOID: observation1, ST: observation1, COUNTY: observation1},
     {GEOID: observation2, ST: observation2, COUNTY: observation2},
     ...,
     {GEOID: observationN, ST: observationN, COUNTY: observationN}]
    }
  5. If your merge was partially (or completely) successful, you’ll be able to export the data as a .csv
    (At the very least, this site can be used as interactive tool to merge data.)
You will now be able to use any of the mapping, plotting, regression table, and monotonic transformation features on your data.
Multiple datasets can be opened and merged as long as they don’t have the same variable names of the EO data or your previously loaded files.

Regression Table:

You can run an multi-variate Ordinary Least-Squares regression using the outcomes and covariates in the EO dataset or between any two variables of any data you may have opened.
  1. Point
  2. Click
No p-values and stars. You’ll have to pay close attention to the value of the coefficient and the size of the standard error.

Plotting:

Some intro
  1. Select the independent (y-axis) and dependent (x-axis) variables and a monotonic transformation that might make sense for your purposes.
    It should plot immediately. Wow! One less click than the regression table.
  2. You can fit the data to a linear or quadratic functional from using the Fit Trend option.
  3. As a default, the scatter plot arithmetic mean values of the dependent and independent variables in 64 mutually exclusive equally sized intervals. The size of each point represents number of observations in a given interval.
    You can choose any number of bins—only positive integer values make sense. If you choose the number of bins to be less than 2 or greater than the number of observations, the scatter plot will load.
